# SPEC.md

個人ブログプロジェクトのスプリント仕様書。スプリント定義とDefinition of Done（DoD）の唯一の事実源。

## 確定済み技術スタック（変更不可）
- Next.js（`output: 'export'` による静的書き出し）
- コンテンツ管理: Markdown/MDX + Velite（Zodスキーマでフロントマターをバリデーション）
- ホスティング: S3 + CloudFront
- CI/CD: GitHub Actions（mainへのpushでビルド→S3同期→CloudFrontキャッシュ無効化）

## コンテンツ構成（確定済み）
- 記事は `content/blog/{YYYY-MM-DD-slug}/index.mdx` に1記事1フォルダで配置し、画像も同フォルダに同梱する
- フロントマター: `title`, `description`, `date`, `tags`, `published`（`published: false` は下書き扱いでビルド時に除外）

## Sprint 0: プロジェクト基盤とMDX表示パイプライン

### 概要
記事を1本書けば `npm run dev` で表示される、ブログの最小骨格を作る。Next.js初期化、Velite導入、MDXレンダリングパイプラインの整備までを行う。タグ一覧・記事一覧ページ・ページネーション・SNS連携など個別機能は対象外（後続スプリントで定義）。

### 機能要件（ユーザーに見える挙動）
- `content/blog/{YYYY-MM-DD-slug}/index.mdx` に置いた記事が、対応するURLでページとして表示される
- 記事ページには見出し（h2/h3等）にアンカーリンクがあり、クリックでその見出しに遷移できる
- 記事内のコードブロックがシンタックスハイライトされた状態で表示される
- 記事に同梱した画像が記事ページ内に表示される
- フロントマターに `published: false` が設定された記事は、`npm run dev` / `npm run build` のどちらでも一覧・個別ページに表示されない
- フロントマターが必須項目（`title`, `description`, `date`, `tags`, `published`）を満たさない記事がある場合、ビルド時にエラーで検知できる
- ページ全体に最低限のベーススタイル（フォント・余白・読みやすい文字幅など）が適用されている

### タスク分解
1. Next.jsプロジェクトの初期化（TypeScript, App Router, Tailwind CSS導入、`output: 'export'` 設定）
2. Velite導入、`content/blog` 配下を対象としたZodスキーマ定義（`title`, `description`, `date`, `tags`, `published` を含む）、`published: false` の記事をビルド出力から除外する設定
3. MDXパイプライン設定: `rehype-pretty-code`（シンタックスハイライト）, `rehype-slug`（見出しID付与）, `rehype-autolink-headings`（見出しアンカーリンク）を組み込む
4. ダミー記事（画像・コードブロック・複数見出しを含む）を1本作成し、表示確認

### Definition of Done
- [ ] `npm run dev` でダミー記事1本が正しく表示される（必須）
- [ ] ダミー記事内の見出しにアンカーリンクが表示され、クリックでその見出しにスクロール/遷移する
- [ ] ダミー記事内のコードブロックがシンタックスハイライトされて表示される
- [ ] ダミー記事に同梱した画像が記事ページ内に表示される
- [ ] `published: false` を設定した記事が一覧・個別ページのどちらにも表示されない
- [ ] フロントマターの必須項目が欠落した記事を置いた状態で `npm run build` を実行するとエラーで失敗する
- [ ] `npm run build` が（不正な記事がない状態で）正常に完了する
- [ ] ページにTailwind CSSによるベーススタイルが適用されている（無装飾のHTMLでない）

## Sprint 1: 記事一覧ページ

### 概要
Sprint 0時点では `app/page.tsx` にダミー記事1本へのリンクのみが表示される最小状態だった。Sprint 1では、複数記事に対応した本来の記事一覧ページに発展させる。一覧の並び順、各記事の表示内容、記事数が多い場合のページネーションを扱う。タグ機能（タグ一覧・タグ別記事一覧）はSprint 2で扱うため、本スプリントのスコープには含めない。

### 機能要件（ユーザーに見える挙動）
- トップページに記事の一覧が表示される（複数記事が存在する状態に対応する）
- 一覧は記事の日付が新しいものから古いものへの順に並ぶ
- 一覧内の各記事には、タイトル・概要（description）・日付が表示される
- 各記事のタイトル（またはカード/項目自体）をクリックすると、その記事の個別ページに遷移する
- 一覧に表示できる記事数が1ページに収まらない場合、複数ページに分割され、ページ間を移動できる（ページネーション）
- ページネーションには現在何ページ目を見ているかが分かる表示があり、前/次のページへ移動する手段がある
- 最初のページ（1ページ目）には「前のページ」に相当する移動手段が無効化または非表示になる、最後のページには「次のページ」に相当する移動手段が無効化または非表示になる
- フロントマターに `published: false` が設定された記事は一覧およびページネーションの対象件数に含まれない（Sprint 0で実装済みのロジックを踏襲し、一覧表示に発展させた後も成立すること）

### タスク分解
1. ダミー記事を、ページネーションが発生する件数まで増やす（日付がそれぞれ異なる複数記事、うち一部は `published: false` を含める）
2. 記事一覧データを日付の新しい順にソートして取得するロジックを実装する
3. トップページ（`app/page.tsx`）を、1件のダミーリンクから複数記事対応の一覧表示に変更する（タイトル・概要・日付を表示）
4. 一覧の件数が1ページの表示上限を超える場合に複数ページへ分割するページネーション機能を実装する（ページ番号付きURLでの遷移、前/次ページ移動、現在ページ表示を含む）
5. `published: false` の記事が一覧・ページネーションの対象件数から除外されることを確認する

### Definition of Done
- [ ] ダミー記事を、1ページの表示上限を超える件数（ページネーションが最低2ページ以上発生する件数）まで用意した状態で `npm run dev` / `npm run build` の両方を確認する
- [ ] トップページの一覧が、各記事の日付に基づき新しい順に並んで表示される
- [ ] 一覧内の各記事に、タイトル・概要（description）・日付が表示されている
- [ ] 一覧内の記事タイトル（またはカード/項目）をクリックすると、対応する記事の個別ページに遷移する
- [ ] 1ページ目を表示した状態で「次のページ」へ移動すると、1ページ目とは異なる記事群が表示され、かつ日付の新しい順という並び順が全ページを通して維持されている
- [ ] 最後のページまで「次のページ」を辿った後、それ以上次のページへ移動できない（ボタンが無効化または非表示になっている）
- [ ] 1ページ目より前のページへは移動できない（「前のページ」に相当する手段が無効化または非表示になっている）
- [ ] ダミー記事の中に `published: false` を設定した記事を含めた状態で、その記事が一覧のどのページにも表示されず、ページネーションの総ページ数・総件数の算出からも除外されている
- [ ] `npm run build` が複数記事・複数ページの状態で正常に完了する

## Sprint 2: タグ機能

### 概要
Sprint 0・Sprint 1では記事の個別表示と日付順の一覧・ページネーションのみを扱った。Sprint 2では、記事に付与された `tags` フロントマターを活用し、タグによる記事の分類・絞り込み・発見を可能にする。記事ページでのタグ表示、タグ別記事一覧ページ（ページネーション含む）、全タグを見渡せるタグ一覧ページを扱う。

### 機能要件（ユーザーに見える挙動）
- 記事の個別ページに、その記事に付与されているタグ（1つまたは複数）が表示される
- 記事が複数のタグを持つ場合、そのすべてのタグが記事ページに表示される
- 記事ページに表示されたタグをクリックすると、そのタグが付与された記事だけを集めた一覧ページに遷移する
- タグ別記事一覧ページでは、該当するタグを持つ記事のみが表示される
- タグ別記事一覧ページの並び順は、Sprint 1の記事一覧と同様に記事の日付が新しいものから古いものへの順である
- タグ別記事一覧ページにおいても、フロントマターに `published: false` が設定された記事は表示されない
- タグ別記事一覧の各記事には、Sprint 1の記事一覧と同様にタイトル・概要（description）・日付が表示される
- タグ別記事一覧の件数が1ページに収まらない場合、Sprint 1と同様にページネーションされ、前/次ページ移動・現在ページ表示・最初/最後のページでの移動手段の無効化（または非表示）が成立する
- すべてのタグを一覧できるタグ一覧ページが存在し、そこから各タグ別記事一覧ページへ遷移できる
- タグ一覧ページには、各タグに対してそのタグが付与された記事の件数（`published: false` の記事を除いた件数）が表示される
- タグ一覧ページに表示される件数や、そもそも一覧に出現するタグは、`published: false` の記事のみが持つタグについては反映されない（その記事を除けば他に同タグの公開記事がない場合、そのタグはタグ一覧に出現しない、または件数0として扱われない）

### タスク分解
1. ダミー記事のタグ付けを見直し、検証に必要な状態を用意する（複数タグを持つ記事、同一タグを複数記事で共有、あるタグ別一覧がページネーションが発生する件数になるよう記事数を調整、あるタグが `published: false` の記事にのみ付与されている状態を含める）
2. 記事の個別ページに、その記事のタグ一覧を表示する機能を実装する（各タグはタグ別記事一覧ページへのリンクになっている）
3. タグ別記事一覧ページを実装する（指定されたタグを持つ公開記事のみを日付の新しい順に取得し、Sprint 1のページネーション機能を流用して表示する）
4. 全タグとその記事数（公開記事のみを対象にカウント）を表示するタグ一覧ページを実装する
5. `published: false` の記事のみが持つタグが、タグ一覧・タグ別記事一覧のいずれにも公開記事として影響を与えないことを確認する

### Definition of Done
- [ ] 複数のタグを持つダミー記事を用意し、その記事の個別ページにすべてのタグが表示されていることを確認する
- [ ] 記事ページに表示されたタグをクリックすると、そのタグのタグ別記事一覧ページに遷移する
- [ ] 同一タグを共有するダミー記事を、1ページの表示上限を超える件数（タグ別一覧でページネーションが最低2ページ以上発生する件数）用意し、そのタグのタグ別記事一覧ページで日付の新しい順に並んで表示されることを確認する
- [ ] タグ別記事一覧ページにおいても、1ページ目で「前のページ」相当の移動手段が無効化または非表示になり、最後のページで「次のページ」相当の移動手段が無効化または非表示になることを確認する
- [ ] タグ別記事一覧の各記事に、タイトル・概要（description）・日付が表示されていることを確認する
- [ ] あるタグを `published: false` の記事にのみ付与した状態を用意し、そのタグがタグ一覧ページに（件数0または存在しない形で）公開記事に影響を与える形で出現しないことを確認する
- [ ] 同一タグが付与された複数記事のうち一部に `published: false` を設定し、そのタグ別記事一覧ページに非公開記事が表示されず、ページネーションの総件数からも除外されていることを確認する
- [ ] タグ一覧ページに、ダミー記事に存在する全タグが表示され、各タグの記事数が実際の公開記事数と一致していることを確認する
- [ ] タグ一覧ページの各タグから対応するタグ別記事一覧ページへ遷移できることを確認する
- [ ] `npm run build` が、複数タグ・タグ別ページネーションが発生するダミー記事の状態で正常に完了する
