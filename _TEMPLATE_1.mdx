{/*
  記事作成テンプレート ─ 使い方メモ

  ■ ファイルの置き場所
  content/blog/YYYY-MM-DD-記事スラッグ/index.mdx
  画像も同じフォルダに入れる（例: cover.jpg）

  ■ frontmatterのルール
  - title       : 記事一覧・記事ページのタイトル
  - description : 一覧ページのタイトル下に出る要約文
  - date        : YYYY-MM-DD形式（一覧は日付降順）
  - tags        : 小文字の英単語推奨。既存タグに寄せると一覧が整理される
                  （これまでの実績: nextjs, aws, deploy, claude-code, mdx, ux, blog）
  - published   : 書きかけの間は false。公開する時に true にして push
                  （false のままなら一覧にもビルドにも出てこない）

  ■ 画像の入れ方
  ![altテキスト](./画像ファイル名.jpg)
  記事フォルダの中に画像を置くだけで、Veliteがビルド時に処理してくれる

  ■ コードブロック
  普通のMarkdownのコードフェンス（```言語名）でOK。
  rehype-pretty-codeが自動でシンタックスハイライトする

  ■ プレビュー
  npm run dev でローカル確認しながら書く

  ■ 公開
  published: true にして git add → commit → push
  GitHub Actionsが自動でビルド・デプロイする

  この下の見出し構成はあくまで一例です。技術手順の記事、日記的な記事、
  トラブルシューティングの記録など、内容に応じて自由に組み替えてください。
  このコメントブロックごと、書き始める前に削除してOKです。
*/}
